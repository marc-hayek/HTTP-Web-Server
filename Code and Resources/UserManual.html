EECE 450 Web Server User Manual:

The EECE 450 Web Server provides you with the ability to retrieve requests conforming with HTTP 1.1 using an internet browser.

The following browsers are identified by the server:
  1. Mozilla Firefox
  2. Microsoft Internet Explorer
  3. Google Chrome
  4. Safari
Starting Up:

To start running the web server, press enter on the welcome screen to enter the server setting screen. Here, the user can set the port number, directory, and the user name and password for protected files. When pleased with the settings, press the run button to start the server. Press Exit to exit the server. Requesting a file:

In order to request a file, enter the ip address of the server followed by a colon ":" which is then followed by the port number(2050 by default) and a forward slash"/".
You can now write the name of the file requested followed by the file type. If a file is not specified, the index page is returned automatically.

Example:

http://192.168.0.1:2050/index.html
http://192.168.0.1:2050/

Requesting a directory:

In order to request a directory, follow the same steps of opening a file but leave type the directory name followed by a forward slash

Example:

http://192.168.0.1:2050/images/

Creating Dynamic Pages:

The web server allows users to create 4 types of dynamic pages:

  1. Dynamic Links from a specific category
  2. Email Link
  3. Dyanmic Images
  4. Mix of links, images, and paragraphs
1.Dynamic Links from a specific category:

The web server allows to users to generate web links that belong to specific category. The available categories are News,Games, and Search. To generate dynamic links, the user must input the following into the URL:

.../category?links=m

The category should be specified from the above mentioned available categories, whereas "m" is an integer from 0 to 10 which represents the number of links requested. The webserver will then randomly generate 10 links of that category. If m is not specified,or is less than 0, 0 links will be generated. If m greater than 10, 10 links will be generated.

2.Email Link:

The web server allows to users to generate an email link specified by the user. To generate an email link, the user must input the following into the URL:

.../email?to=fullemailaddress

The server will then generate a link to that email address.

3.Dynamic Images:

The web server allows to users to generate a specified number of images. Each image generated also contains a link corresponding to that image. Clicking on the image would open the link. To generate a set of images, the user must input the following into the URL:

.../images?number=n

The number n should be a positive number, and it specifies the number of images requested. The web sever will generate a random set of images.

4.Mix of links, images, and paragraphs:

The web server allows users to generate a mix of links,images, and paragraphs. To generate the links,images,and paragraphs, the user must input the following into the URL in any order:

.../mix?paragraphs=m,links=m,images=n

The integers specified follow the same rules as mentioned above. The web sever will generate a random set of images, links, and paragraphs.
